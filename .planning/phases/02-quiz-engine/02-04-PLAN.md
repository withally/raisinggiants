  <action>
    Create the top-level quiz orchestrator. This is the most complex component in Phase 2 — it wires everything together.

    **'use client' component** with the following responsibilities:

    **1. Zustand hydration check:**
    ```typescript
    const hasHydrated = useQuizStore.persist.hasHydrated()
    if (!hasHydrated) return <div className="min-h-screen" />  // blank while rehydrating
    ```
    Prevents flash of step-0 for returning users.

    **2. nuqs step synchronization (QUIZ-03 — browser Back button):**
    ```typescript
    import { useQueryState } from 'nuqs'

    const [step, setStep] = useQueryState('step', {
      parse: (v) => parseInt(v) || 0,
      serialize: String,
      defaultValue: 0,
      history: 'push',   // CRITICAL: enables browser Back button between questions
    })
    ```
    - nuqs is the source of truth for which step to render
    - On mount: if URL has `?step=N`, use N. If no step param, sync nuqs to Zustand store's `currentStep` (for localStorage resumption)
    - When step changes via nuqs (browser Back), update Zustand store direction to 'back'

    **IMPORTANT:** nuqs requires a `NuqsAdapter` provider. Wrap the quiz page (or the root layout) with the Next.js App Router adapter:
    ```typescript
    // In app/quiz/page.tsx or app/layout.tsx:
    import { NuqsAdapter } from 'nuqs/adapters/next/app'
    ```

    **3. Anonymous Supabase session init:**
    ```typescript
    useEffect(() => {
      async function initSession() {
        const store = useQuizStore.getState()
        const supabase = createClient()

        if (store.userId) {
          // Returning user — check if session is still valid
          const { data: { session } } = await supabase.auth.getSession()
          if (!session) {
            // Expired — create new anonymous session
            const { data: { user } } = await supabase.auth.signInAnonymously()
            if (!user) return // show error state
            store.setUserId(user.id)
            // Create new quiz session row for new user
            const { data: newSession } = await supabase
              .from('quiz_sessions')
              .insert({ user_id: user.id, status: 'in_progress' })
              .select('id')
              .single()
            if (newSession) store.setSessionId(newSession.id)
          }
          return
        }

        // First-time user
        const { data: { user } } = await supabase.auth.signInAnonymously()
        if (!user) return // show error/retry state
        store.setUserId(user.id)

        const { data: session } = await supabase
          .from('quiz_sessions')
          .insert({ user_id: user.id, status: 'in_progress' })
          .select('id')
          .single()
        if (session) store.setSessionId(session.id)
      }

      initSession()
    }, [])
    ```

    **4. Step routing (rendering logic):**
    ```typescript
    const totalQuestions = QUESTIONS.length
    // step 0 to totalQuestions-1: quiz questions
    // step totalQuestions: email capture screen
    // step totalQuestions+1: processing screen

    if (step < totalQuestions) {
      return <QuizCard question={QUESTIONS[step]} ... />
    } else if (step === totalQuestions) {
      return <EmailCaptureScreen onSubmit={onQuizComplete} isSubmitting={isSubmitting} />
    } else {
      return <ProcessingScreen />
    }
    ```

    **5. Auto-advance on answer selection:**
    When user taps an option:
    1. Set answer in Zustand store: `store.setAnswer(questionId, answerId)`
    2. Wait 300ms (visible selected state)
    3. Advance step: `setStep(step + 1)` via nuqs (which creates a browser history entry)
    4. Update Zustand direction: `store.goForward(step + 1)`

    ```typescript
    const handleAnswer = useCallback((questionId: string, answerId: string) => {
      const store = useQuizStore.getState()
      store.setAnswer(questionId, answerId)
      setTimeout(() => {
        const nextStep = step + 1
        setStep(nextStep)
        store.goForward(nextStep)
      }, 300)
    }, [step, setStep])
    ```

    **6. Back button handler:**
    Top-left arrow icon (hidden on step 0). On click:
    ```typescript
    const handleBack = useCallback(() => {
      if (step > 0) {
        const prevStep = step - 1
        setStep(prevStep)
        useQuizStore.getState().goBack(prevStep)
      }
    }, [step, setStep])
    ```

    **7. Slide transitions:**
    Use CSS keyframes for slide left (forward) and slide right (back) animations:
    ```css
    @keyframes slideInRight { from { transform: translateX(100%); opacity: 0; } to { transform: translateX(0); opacity: 1; } }
    @keyframes slideInLeft { from { transform: translateX(-100%); opacity: 0; } to { transform: translateX(0); opacity: 1; } }
    ```
    Apply the appropriate animation based on `store.direction`:
    ```tsx
    <div key={step} className={direction === 'forward' ? 'animate-slide-in-right' : 'animate-slide-in-left'}>
    ```
    Define the keyframe animations in app/globals.css or as Tailwind utilities.

    **8. Quiz completion flow (`onQuizComplete`):**
    ```typescript
    async function onQuizComplete(email: string) {
      setIsSubmitting(true)
      const store = useQuizStore.getState()
      store.setEmail(email)

      // Step 1: Compute dimension profile
      const dimensionProfile = computeDimensionProfile(store.answers)

      // Step 2: Get archetype result
      const result = getResult(dimensionProfile)

      // Step 3: Extract cultural background
      const culturalBackground = store.answers['q-cultural-background'] ?? null

      // Step 4: Show processing screen immediately
      setStep(totalQuestions + 1)

      // Step 5: Update quiz_sessions row with completed data
      const supabase = createClient()
      const { error } = await supabase
        .from('quiz_sessions')
        .update({
          status: 'completed',
          answers: store.answers,
          dimension_scores: dimensionProfile,
          archetype_id: result.primary,
          cultural_background: culturalBackground,
          email: email,
          completed_at: new Date().toISOString(),
        })
        .eq('id', store.sessionId)

      if (error) {
        // Handle error — show retry or error state
        console.error('Failed to save quiz results:', error)
        setIsSubmitting(false)
        return
      }

      // Step 6: Wait minimum 2-3 seconds for processing screen feel
      await new Promise(resolve => setTimeout(resolve, 2500))

      // Step 7: Redirect to result page
      // Do NOT call store.reset() here — Phase 3 may need localStorage data
      router.push(`/result?session=${store.sessionId}`)
    }
    ```

    **9. Section header detection:**
    Determine if the current question is the first in its section:
    ```typescript
    const showSectionHeader = step === 0 || QUESTIONS[step].section !== QUESTIONS[step - 1].section
    ```
    Pass this to QuizCard.

    **10. Progress bar integration:**
    Render `<QuizProgress current={step} total={totalQuestions} />` above the card content. The progress bar is fixed at the top of the screen.

    **11. Error handling:**
    If Supabase session init fails (signInAnonymously returns error), show a simple error state with a retry button. Do not silently lose progress.
