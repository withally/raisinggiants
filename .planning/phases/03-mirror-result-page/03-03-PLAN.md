**Create `components/result/EmailGateOverlay.tsx`:**
- Client Component ("use client")
- Named export: `export function EmailGateOverlay({ onDismiss }: { onDismiss: () => void })`
- Fixed overlay: `position: fixed; inset: 0; z-index: 50` with `bg-amber-50/95 backdrop-blur-sm`
- Centered content card (matching EmailCaptureScreen design language from quiz):
  - Headline: "Your result is ready" in display font (Cormorant Garamond)
  - Sub-copy: "We've prepared a personalised report about the parenting you received. Enter your email to reveal it."
  - Email form using `react-hook-form` + `zod` (same pattern as `EmailCaptureScreen.tsx`):
    - Email input with label and aria-describedby for errors
    - Submit button: "Reveal My Result"
    - Disabled state during submission
  - Privacy note below the form
- On valid email submit: call `onDismiss()` — the parent (ResultPageClient) handles the state transition
- Gate is purely UI — NO Supabase write. The email was already captured during quiz completion.
- Accessibility: `role="dialog"`, `aria-modal="true"`, `aria-label="Reveal your result"`
- Focus management: auto-focus the email input on mount (use `autoFocus` prop on Input)
- The form submission should NOT do any network call — just validate email format and call onDismiss

**Create `components/result/ResultPageClient.tsx`:**
- Client Component ("use client")
- Named export: `export function ResultPageClient({ children }: { children: React.ReactNode })`
- State: `const [gateOpen, setGateOpen] = useState(true)` — initial true for SSR safety (content starts blurred)
- State: `const [transitioning, setTransitioning] = useState(false)` — for animation state
- When `gateOpen` is true:
  - Children wrapper div has: `filter: blur(12px)`, `pointer-events: none`, `user-select: none`, `aria-hidden: true`
  - `<EmailGateOverlay onDismiss={handleDismiss} />` renders on top
  - `<StickyBlueprintBar hidden={true} />` is hidden
- `handleDismiss` function:
  1. Set `transitioning = true`
  2. Set `gateOpen = false`
  3. After 800ms (transition duration), set `transitioning = false`
- When `gateOpen` is false and `transitioning` is true:
  - Children wrapper has `animate-content-unblur` class
  - Overlay gets `animate-gate-fade-out` class + `pointer-events: none`
- When `gateOpen` is false and `transitioning` is false:
  - Children render normally (no blur, no overlay)
  - `<StickyBlueprintBar hidden={false} />` is visible
- CSS transition for blur: `transition: filter 0.8s ease` on the content wrapper as a fallback

**Update `app/result/page.tsx`:**
- Import `ResultPageClient` from `components/result/ResultPageClient`
- Wrap the existing content (ArchetypeReveal, FoundationalPatternsSection, WatchoutsSection, BlueprintCTA, CulturalSection) inside `<ResultPageClient>...</ResultPageClient>`
- The Server Component renders all content into the DOM (for SSR), and the Client wrapper controls visibility via blur/overlay
- Ensure the error states (no session, invalid session) are NOT wrapped in ResultPageClient — errors show directly without a gate
