**Step 1: Audit current codebase for brand-affecting files.**
Run these commands to discover all files that need changes:

```bash
# Find all files with hardcoded amber/stone color classes

# Find illustration/icon files
ls components/landing/illustrations/

# Find public assets
ls public/
```

**Step 2: Create file-by-file change matrix.**

For each brand-affecting file, document:
- File path
- What it currently controls (from RESEARCH.md codebase analysis)
- What changes per the visual direction brief (06-04)
- Which implementation phase it belongs to (A, B, or C)
- Estimated effort category: trivial (find-replace), moderate (restructure), significant (redesign/recreate)

**Step 3: Organize into three implementation phases.**

**Phase A: Foundation (CSS tokens + fonts)**
Files: `app/globals.css`, `app/layout.tsx`
What changes: CSS custom property values (palette tokens), font imports, metadata strings
Why first: All component color changes depend on updated tokens; this is the lowest-risk, highest-impact change
Estimated scope: 1 plan, ~15 min

**Phase B: Component Color Migration**
Files: All component files with hardcoded amber/stone Tailwind classes
What changes: Replace hardcoded color classes with new palette classes (e.g., `amber-500` -> the new accent class)
Why second: Depends on Phase A tokens being in place; systematic find-replace across 11+ component files
Estimated scope: 1-2 plans, ~30 min

**Phase C: Asset Replacement (illustrations, icons, public assets)**
Files: `MirrorIllustration.tsx`, `ArchetypeIcons.tsx`, `public/` assets (logo, OG image, favicon)
What changes: New illustrations matching the visual direction brief imagery system; new brand assets
Why third: Most effort, requires design decisions at implementation time (PNG replacement vs inline SVG)
Estimated scope: 1-2 plans, ~45 min

**Step 4: Brand name technical checklist (if name change recommended).**

If 06-03 recommended a name change, add a section covering:
- Domain acquisition (user action)

**Step 5: Summary and recommendations.**
- Total estimated implementation effort (across all three phases)
- Recommended execution order with rationale
- Dependencies between phases
- Risk assessment: what could go wrong in each phase and how to mitigate

**CRITICAL: This plan produces the execution plan document, NOT the actual rebrand implementation.** Phase 6 ends at the brief and the plan. Implementation is a separate future phase.

**Do NOT:**

- Make implementation decisions that the execution plan should leave to the implementation phase (e.g., the specific illustration style choice is in the visual direction brief, but whether to use PNG replacement or inline SVG is an implementation-time decision)
