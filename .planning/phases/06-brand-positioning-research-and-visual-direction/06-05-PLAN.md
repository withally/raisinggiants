### Section 2: What gets REBUILT (component layer)

For each of the 8 landing page sections, create a component rebuild spec that translates the 06-04 design spec into developer instructions:

**For each component, document:**

1. **File:** `components/landing/[Name].tsx`
2. **Current state (one sentence):** What the template version does
3. **Rebuild target:** What the 06-04 design spec says this section should be (reference the specific spec)
4. **Keep from current:** Any data/content that transfers (researcher data, FAQ content, archetype names, quote data, copy)
5. **Rewrite:** The entire component JSX/layout
6. **Key implementation details:** Specific Tailwind classes, layout approach, responsive behavior, any tricky parts
7. **New dependencies:** Any new hand-drawn components or utilities needed
8. **Responsive notes:** How it works at mobile (375px), tablet, and desktop (1440px)

Example level of detail needed for one component:
```
HeroSection.tsx:
- CURRENT: 5 competing elements in max-w-6xl container, centered composition
- REBUILD: Full-viewport typographic composition. Headline at text-[200px] xl scale, flush-left.
  Below headline: one-line subtitle + CTA inline. Nothing else.
  Supporting copy, research badge, eyebrow label — ALL REMOVED from hero (move to other sections or delete).
  Background: grain texture on Cloud White. No radial gradient blob.
  Desktop: headline left-aligned, right 40% of viewport is empty space.
  Mobile: headline stacks naturally at text-6xl, CTA full-width below.
- KEEP: The headline copy ("Finally, the words for it."), the CTA destination (/quiz)
- REWRITE: All JSX — completely different spatial composition
```

Do this for ALL 8 components.

### Section 4: Implementation waves

Organize the rebuild into logical waves. Each wave should be independently testable.

**Wave 1: Foundation additions (~15 min)**
- Any new hand-drawn components in hand-drawn.tsx
- Any new CSS utilities in globals.css
- Any globals.css refinements (if any — it's mostly complete)

**Wave 2: Hero + Social Proof rebuild (~30 min)**
- Rewrite HeroSection.tsx from scratch per 06-04 spec
- Rewrite SocialProof.tsx from scratch per 06-04 spec
- These are the first two sections users see — test the new direction immediately

**Wave 3: Middle sections rebuild (~45 min)**
- Rewrite ArchetypePreview.tsx per 06-04 spec
- Rewrite KOLCredibility.tsx per 06-04 spec
- Rewrite KOLNarrative.tsx per 06-04 spec
- These are the content-dense sections — the bento layouts, immersive backgrounds

**Wave 4: Closing sections + page composition (~20 min)**
- Rewrite FAQ.tsx per 06-04 spec
- Rewrite SecondaryHero.tsx per 06-04 spec
- Rewrite Footer.tsx per 06-04 spec
- Update page.tsx if section order changes
- Final scroll-through verification

**Wave 5: Visual checkpoint — user approval**
- Screenshot full page at 1440px desktop
- Screenshot full page at 375px mobile
- Present to user for approval
- Does it feel like "a beautifully designed book that lives on the web"?
- Does each section feel different from its neighbors?
