Create `scripts/validate-scoring.ts` — a comprehensive validation script that tests the scoring system against diverse simulated answer profiles.

Import from:
- `../lib/quiz/scoring-matrix.ts`: `scoreArchetypes`, `getResult`, `ARCHETYPE_PROFILES`, `DIMENSION_WEIGHTS`
- `../lib/archetypes/archetypes.ts`: `ARCHETYPES`, `validateArchetypeDistinctness`
- `../lib/quiz/dimensions.ts`: `DIMENSION_KEYS`

**Section 1: Simulated Profiles (20-30 profiles)**

Design profiles to cover the diversity requirements from RESEARCH.md Pitfall #4:

Category A — Classic Patterns (8-10 profiles):
- High warmth + high boundaries + high regulation (authoritative-adjacent)
- High boundaries + low warmth + low autonomy (authoritarian-adjacent)
- High warmth + low boundaries + high permissiveness (permissive-adjacent)
- High protective instinct + low autonomy + moderate warmth (overprotective)
- High presence + high emotional coaching + high warmth (conscious/attuned)
- High narrative coherence + moderate across others (the "earned secure" profile)
- Low warmth + low boundaries + low engagement (disengaged profile)
- High warmth + high autonomy + low boundaries (free-range adjacent)

Category B — Edge Cases (6-8 profiles):
- ALL dimensions at 5 (the indeterminate / flat case)
- Single dimension maxed at 10, all others at 3 (one-dimensional profiles — one for each of the 3 most important dimensions)
- ALL dimensions at 9 (the "everything high" ceiling case)
- ALL dimensions at 2 (the "everything low" floor case)
- High protective instinct (10) + low emotional regulation (2) — trauma-suggestive pattern
- High narrative coherence (10) + low warmth (3) — intellectualized but emotionally disconnected

Category C — Culturally Diverse Patterns (4-6 profiles):
- High boundary consistency + high warmth + low autonomy support (collectivist-adjacent pattern)
- High autonomy support + moderate warmth + low boundary consistency (Western permissive-progressive)
- High protective instinct + high warmth + high boundary consistency (South Asian joint-family pattern)
- Moderate across all with high presence (mindful parenting pattern, culturally neutral)
- High boundary consistency + low warmth + high protective instinct (strict disciplinarian cross-cultural)

Category D — Blended / Ambiguous (4-6 profiles):
- Two archetypes should score nearly equally (test that primary vs. secondary is a close call)
- Profile that sits equidistant from 3+ archetypes (maximally ambiguous)
- Profile that strongly matches one archetype but has one extreme outlier dimension
- Profile with high variance (some dimensions at 9, others at 2) vs. low variance (all dimensions 5-7)

Each profile should have a `label` (descriptive name), `profile` (DimensionProfile), and `expectedBehavior` (what we expect: which archetype should be primary, or "should be ambiguous", or "should not be X").

**Section 2: Run Scoring and Collect Results**

For each simulated profile:
1. Call `getResult(profile)` to get primary + 3 secondaries
2. Call `scoreArchetypes(profile)` to get full ranked list with scores
3. Record: profile label, primary archetype, primary score, score gap between primary and second (indicator of confidence), all 4 ranked archetype ids

**Section 3: Analysis Output**

Print to console:

1. **Per-profile results table:**
   ```
   | Profile Label | Primary | Score | Gap to 2nd | Secondary 1-3 | Expected | Match? |
   ```

2. **Distribution analysis:**
   - Count how many profiles map to each archetype as primary
   - Flag if any archetype captures >40% of profiles (degenerate)
   - Flag if any archetype captures 0 profiles (dead archetype)

3. **Confidence analysis:**
   - Average gap between primary and secondary scores
   - Flag profiles where gap < 0.05 (very ambiguous — not necessarily bad, but worth noting)
   - Flag profiles where primary score < 0.6 (poor match to any archetype)

4. **Distinctness validation:**
   - Call `validateArchetypeDistinctness()` and print result
   - If any archetype pair fails the 3-point/2-dimension rule, print which pair and which dimensions are too close

5. **Edge case report:**
   - Did the indeterminate case produce a result? (It should — there is no "no match" state)
   - Did single-dimension-maxed profiles produce different primary archetypes? (They should not all map to the same one)
   - Did the trauma-suggestive profile map to a clinically plausible archetype?

**Section 4: Pass/Fail Summary**

Print a final PASS/FAIL verdict based on:
- [ ] All 20-30 profiles produced results (no crashes)
- [ ] No archetype captures >40% of profiles
- [ ] No archetype captures 0 profiles
- [ ] Average confidence gap > 0.03
- [ ] Distinctness validation passes
- [ ] Indeterminate case produces a result

If FAIL: print specific recommendations for which archetype profiles to adjust.

**Execution:** The script should run with `npx ts-node scripts/validate-scoring.ts` or `npx tsx scripts/validate-scoring.ts`. Use console.log for all output. Do not require any external dependencies beyond ts-node/tsx.

**IMPORTANT:** After creating the script, RUN IT. Capture the output. If any failures occur, adjust the archetype dimension profiles in `lib/archetypes/archetypes.ts` and re-run until PASS. This task is not done until the validation passes.
