Create a comprehensive review document at `.planning/phases/00-archetype-framework/sophia-review-package.md` that compiles the complete archetype framework into a format optimized for Sophia's review.

**Document structure:**

1. **Executive Summary** — 3-4 sentences explaining what is being reviewed and what Sophia needs to validate

2. **Scoring Dimensions** — List all 6-7 dimensions with:
   - Name and description
   - Low/high labels
   - Which KOL research anchors each dimension
   - Review prompt: "Are these the right dimensions? Should any be added, removed, or renamed?"

3. **Archetype Overview Table** — For each archetype:
   | Archetype Name | Tagline | Key High Dimensions | Key Low Dimensions |
   Followed by review prompt: "Do these archetypes feel clinically distinct? Are any too similar? Is anything missing?"

4. **Full Archetype Profiles** — For each archetype:
   - Name and tagline
   - Dimension profile (visual bar chart using Unicode: `emotionalWarmth: 9 ████████░`)
   - Foundational patterns (headline + themes)
   - Watchouts (headline + themes)
   - Review prompt per archetype: "Does this feel clinically accurate? Are the watchouts fair and normalized?"

5. **Scoring Simulation Results** — Paste the output from Plan 03's validation script:
   - Per-profile results table
   - Distribution analysis
   - Edge case handling
   - Review prompt: "Do these scoring results feel clinically plausible? Would a parent with [profile X] likely identify with [archetype Y]?"

6. **Cultural Overlay Samples** — Show 2-3 complete overlay examples (not all 35):
   - Pick the archetype with the most interesting cultural variation
   - Show all 4-5 cultural contexts for that archetype
   - Review prompt: "Are these cultural modifiers respectful and accurate? Are there cultural contexts we should add?"

7. **Sign-Off Checklist** — Clear yes/no items for Sophia:
   - [ ] Dimensions are clinically sound
   - [ ] Archetype set is complete (no missing patterns, no redundant archetypes)
   - [ ] Archetype names are warm and non-pathologizing
   - [ ] Foundational patterns are accurate
   - [ ] Watchouts are fair and normalized
   - [ ] Scoring simulation results are clinically plausible
   - [ ] Cultural overlays are respectful and accurate
   - [ ] Framework is ready for engineering (Phase 1+)

8. **Revision Notes Section** — Empty section for Sophia to write adjustment requests

**Tone:** Professional but warm. Sophia is a clinical expert reviewing a product framework, not a peer reviewer reading an academic paper. Make it easy for her to say "yes, this is right" or "change X to Y."

**Read all archetype data** from:
- `lib/archetypes/archetypes.ts` (names, taglines, profiles, patterns, watchouts)
- `lib/archetypes/cultural-overlays.ts` (overlay data)
- `lib/quiz/dimensions.ts` (dimension definitions)
- `lib/quiz/scoring-matrix.ts` (dimension weights)
- Run `npx tsx scripts/validate-scoring.ts` and capture output for the simulation results section
