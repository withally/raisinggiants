4. `DimensionKey` — string literal union of dimension identifiers
5. `DimensionDefinition` — metadata for each dimension (name, description, KOL anchors, min/max labels)
6. `ArchetypeContent` — structured outline with headline, themes (3-5 bullets), researchAnchor, citations
7. `Citation` — researcher name, work title, year, relevance note
8. `CulturalOverlay` — culturalContext, expressionModifier, strengthsInContext, tensionsInContext

- `DimensionProfile` must use `Record<DimensionKey, number>` so adding/removing dimensions does not require type changes

- Do NOT include any runtime values — this file is types only

**Part A: KOL Synthesis Document (.planning/phases/00-archetype-framework/kol-synthesis.md)**

Create a structured synthesis document that cross-references major parenting researchers' frameworks to identify the dimension space. This is the bedrock research artifact that justifies the chosen scoring dimensions.

Structure:
1. **KOL Synthesis Table** — For each of the following researchers, document: framework name, key dimensions contributed, validated instruments (if any), and relevance to the product's three-lens model:
   - Diana Baumrind (Parenting Styles: Responsiveness, Demandingness)
   - Maccoby & Martin (Extended 4-quadrant model)
   - John Gottman (Emotional Coaching: Emotional Awareness, Empathy, Coaching vs. Dismissing)
   - Daniel Siegel (Interpersonal Neurobiology: Mindsight, Emotional Integration, Narrative Coherence)
   - Mary Ainsworth (Attachment: Secure/Anxious/Avoidant/Disorganized)
   - Mary Main (Adult Attachment Interview: Coherent Narrative About Past, Earned Secure)
   - Shefali Tsabary (Conscious Parenting: Presence vs. Projection)
   - Siegel & Hartzell (Parenting from the Inside Out: Autobiographical Narrative Integration)
   - PBDQ research / Loeber et al. (6-factor model)
   - Additional: Lansford et al. (cross-cultural), van IJzendoorn (intergenerational transmission)

2. **Dimension Overlap Matrix** — A table showing which dimensions appear across multiple frameworks (merged under a common name where different researchers use different terms for the same construct)

3. **Recommended Dimensions** — The final 6-7 dimensions selected for the scoring system, with justification for each

4. **Excluded Dimensions** — Dimensions considered but excluded, with reasons (e.g., clinically valid but cannot be measured via quiz questions)

**Part B: Dimension Definitions (lib/quiz/dimensions.ts)**

Create `lib/quiz/dimensions.ts` that defines the scoring dimensions as runtime data:

1. Import `DimensionKey` and `DimensionDefinition` from `../archetypes/types.ts`
2. Define a `DIMENSION_KEYS` const array of all dimension identifiers (e.g., `['emotional-warmth', 'boundary-consistency', ...]`)
3. Define a `DIMENSIONS` array of `DimensionDefinition` objects, one per dimension, each containing:
   - `key`: the DimensionKey identifier
   - `name`: human-readable name (e.g., "Emotional Warmth")
   - `description`: 2-3 sentence description of what this dimension measures
   - `lowLabel`: what a low score means (e.g., "Emotionally reserved, affection expressed through action rather than words")
   - `highLabel`: what a high score means (e.g., "Openly affectionate, verbally expressive of love and validation")
   - `researchAnchors`: array of KOL names/frameworks this dimension derives from
   - `candidateQuestions`: 2-3 example quiz question ideas that would measure this dimension (per research pitfall #2 — validate dimensions can survive quiz translation)
   - `lensNotes`: brief note on whether this dimension measures differently across Past/Current/Aspirational lenses

Use the 6-7 dimensions recommended in RESEARCH.md as the starting point:
1. Emotional Warmth (Baumrind responsiveness, Gottman empathy, PBDQ factor 1)
2. Boundary Consistency (Baumrind demandingness, authoritarian dimension)
3. Autonomy Support (Baumrind, PBDQ factor 4)
4. Emotional Regulation / Coaching (Gottman, Siegel)
5. Protective Instinct / Vigilance (PBDQ anxious intrusiveness factor 3)
6. Narrative Coherence about Own Past (Siegel, Main's AAI)
7. Presence / Attunement (Tsabary, Siegel)

For each dimension, write 2-3 candidate quiz questions per the research anti-pattern guidance ("if no natural question can be written that doesn't feel clinical or leading, the dimension may need to be operationalized differently"). If a dimension is hard to operationalize, note this clearly.

Export `DIMENSION_KEYS` and `DIMENSIONS`.
