Create `lib/archetypes/archetypes.ts` defining the complete set of parenting archetypes. Import types from `./types.ts` and dimension data from `../quiz/dimensions.ts`.

**Archetype design constraints (from user decisions):**
- **Research-driven count** — let the dimension profiles produce natural clusters. Target 6-8 but do not force-fit.
- **Warm, descriptive names** referencing current parenting style (NOT childhood origin, NOT clinical terms).
  - Good: "The Steady Anchor", "The Fierce Protector", "The Gentle Guide"
  - Bad: "The Anxious Parent", "Avoidant Type", "The Wounded Healer"
- **Spectrum with overlap** — archetypes shade into each other. Adjacent archetypes share some high dimensions but differ on at least 2 dimensions by at least 3 points.
- **Strength-first** — every archetype name and tagline must pass the test: "Would a parent proudly share this as their result?"
- **One-liner shareable tagline** per archetype (e.g., "Your calm is your children's foundation")

**For each archetype, define:**
1. `id` — kebab-case identifier (e.g., "steady-anchor")
2. `name` — warm name (e.g., "The Steady Anchor")
3. `tagline` — one-liner shareable (max 10 words)
4. `dimensionProfile` — scores (1-10) on each dimension from dimensions.ts. Design profiles so each archetype has a DISTINCT signature — at least 2 dimensions differ by 3+ points from every other archetype.
5. `foundationalPatterns` — set to empty placeholder: `{ headline: '', themes: [], researchAnchor: '', citations: [] }`
6. `watchouts` — set to empty placeholder: `{ headline: '', themes: [], researchAnchor: '', citations: [] }`
7. `culturalOverlays` — set to empty array: `[]`

**Design approach:**
- Start from the KOL synthesis and dimension definitions in Plan 01 outputs
- Map the well-known parenting patterns to dimension profiles:
  - High warmth + high boundaries + high regulation = authoritative-adjacent (the "gold standard" profile)
  - High boundaries + low warmth + low autonomy = authoritarian-adjacent
  - High warmth + low boundaries = permissive-adjacent
  - High protective instinct + low autonomy support = overprotective pattern
  - High presence + high emotional coaching = conscious/attuned pattern
  - High narrative coherence + earned secure = healing/growth pattern
  - And 1-2 more that emerge from the dimension space
- BUT do not name them after clinical categories. Transform each into a warm, identity-positive name.

**Required exports:**

1. `ARCHETYPE_PROFILES` — a `Record<ArchetypeId, DimensionProfile>` extracted from the ARCHETYPES array (for direct lookup during scoring)

2. `DIMENSION_WEIGHTS` — a `Partial<Record<DimensionKey, number>>` defining relative importance weights for each dimension. Default all to 1.0. 3. `scoreArchetypes(userProfile: DimensionProfile, weights?: Partial<Record<DimensionKey, number>>): ScoredArchetype[]` — the core scoring function:
   - For each archetype, calculate weighted similarity: `sum of (1 - |user_dim - archetype_dim| / 9) * weight` for each dimension, normalized by total weight
   - Return sorted array (descending by score) of `{ archetypeId: ArchetypeId, score: number }` where score is 0-1
   - Use the `weights` parameter if provided, otherwise fall back to `DIMENSION_WEIGHTS`

4. `getResult(userProfile: DimensionProfile, weights?: ...): { primary: ArchetypeId, secondaries: [ArchetypeId, ArchetypeId, ArchetypeId] }` — convenience function that returns the top-4 ranked archetypes in the 1-primary + 3-secondary format per user decision

5. `softInferCurrentFromPast(pastResult: { primary: ArchetypeId, secondaries: ArchetypeId[] }): LensResult` — stub implementation for soft inference. For now, return the same archetype ranking with `isSoftInference: true` and `completedAt: null`. Include a TODO comment: "Replace with empirical mapping table after launch data collection. Current implementation assumes intergenerational transmission (van IJzendoorn 75% secure-secure transmission)."

**Type definition:**
```typescript
interface ScoredArchetype {
  archetypeId: ArchetypeId;
  score: number; // 0-1 normalized similarity
}
```

**Important:** Use the normalized distance formula from RESEARCH.md (not cosine similarity). The max delta is 9 (scale 1-10). Include JSDoc comments explaining the algorithm.
