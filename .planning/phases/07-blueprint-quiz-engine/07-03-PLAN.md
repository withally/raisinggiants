    **Email Verify API** (`app/api/bp-verify-email/route.ts`):
    POST endpoint that verifies an email against a Blueprint session:
    - Accept `{ sessionId, email }` in request body
    - Use `createAdminClient()` to fetch `bp_quiz_sessions` row by `id = sessionId`
    - Compare provided email (case-insensitive, trimmed) against stored `email`
    - If match: return `{ verified: true }`
    - If no match: return `{ verified: false }` with 403 status
    - If session not found: return error with 404
    - IMPORTANT: Do NOT put the email in the URL (per RESEARCH.md anti-pattern). This is a POST endpoint.

    **BlueprintEmailGate** (`components/blueprint-result/BlueprintEmailGate.tsx`):
    "use client" inline form component that gates the result content.

    Props: `{ sessionId: string, onVerified: () => void }`

    UI:
    - Centered card on cream background (#FAFAF7)
    - Headline: "Enter your email to view your results"
    - Subtext: "We use this to verify your access. Enter the email you used when you took the quiz."
    - Email input (text type, autocomplete=email)
    - Submit button with teal bg (#0F5845)
    - Error state: "That email doesn't match our records" (on 403)
    - Loading state: button disabled with "Verifying..." text

    Behavior:
    - On submit: POST to `/api/bp-verify-email` with `{ sessionId, email }`
    - On success (verified: true): call `onVerified()` callback
    - On failure (verified: false / 403): show error message
    - No retry limit (keep it simple for Phase 7)

    The parent component (BlueprintResultClient or the page itself) tracks `isVerified` state. When false, only the gate shows. When true, the full result renders.

    **Create BlueprintResultClient** wrapper in `app/blueprint/result/page.tsx` (or as a separate component file if needed for the "use client" boundary). This Client Component:
    - Receives all result data as props (archetype, patterns, watchouts, cultural overlay, mirror archetype for bridge)
    - Manages `isVerified` state (default false)
    - Renders `<BlueprintEmailGate>` when not verified
    - Renders all result sections when verified
    - NOTE: If adding a separate file for BlueprintResultClient, add it to `components/blueprint-result/BlueprintResultClient.tsx` and update the files list above
