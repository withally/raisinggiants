Create `app/result/page.tsx` as a Server Component following the same pattern as `app/quiz/page.tsx`:
- Export metadata: `{ title: "Your Parenting Archetype | Your Parenting Blueprint" }`
- Wrap `ResultShell` in a `<Suspense>` boundary (useSearchParams requires it)
- Suspense fallback: `<div className="min-h-screen bg-amber-50" />`

Create `components/result/ResultShell.tsx` as a `"use client"` component:
- Use `useSearchParams()` to get `session` param
- Fetch the quiz_sessions row from Supabase using `createClient()` from `@/lib/supabase/client`:
  ```ts
  const { data, error } = await supabase
    .from('quiz_sessions')
    .select('archetype_id, cultural_background, dimension_scores, answers')
    .eq('id', sessionId)
    .single()
  ```
- While loading, read Zustand `useQuizStore` for optimistic archetype data (if sessionId matches store.sessionId, use store data to render skeleton with archetype name while fetch completes — SSR-safe hydration pattern from 02-04: useState(false) + useEffect + onFinishHydration)
- On successful fetch: look up primary archetype via `ARCHETYPES.find(a => a.id === data.archetype_id)`
- Compute secondary archetypes: call `computeDimensionProfile()` on the stored answers to reconstruct the DimensionProfile, then call `getResult()` to get `{ primary, secondaries }`. Use `secondaries[0]` for the blend mention. Note: the DB stores `dimension_scores` JSONB directly, so alternatively cast it as DimensionProfile and pass to `getResult()`. Use the DB `dimension_scores` directly (it IS the DimensionProfile already computed at quiz completion) — no need to recompute from answers.
- Look up cultural overlay: `archetype.culturalOverlays.find(o => o.culturalContext.toLowerCase().includes(data.cultural_background))` — may be null if cultural_background was not provided.
- Pass all resolved data down to child components: `ResultHero`, and later teaser sections (render only `ResultHero` for now; Plan 03-02 adds teaser sections).
- Error states: missing session param, fetch error, null archetype_id, archetype not found in ARCHETYPES → all render `ResultError`.
- Use warm amber-50 background (`bg-amber-50 min-h-screen`) consistent with quiz.

Create `components/result/ResultSkeleton.tsx`:
- Simple loading UI: `min-h-screen bg-amber-50` with centered animated dots (same pattern as ProcessingScreen: three amber dots with staggered animate-bounce).
- No shimmer required per RESEARCH.md decision.

Create `components/result/ResultError.tsx`:
- Display "We couldn't find your results" message
- Link to `/quiz` to retake: "Try retaking the quiz"
- Warm styling consistent with amber-50 theme.

Run `npx biome check --write app/result/ components/result/` after creating files.

Create `components/result/ResultHero.tsx` as a client component receiving props:
```ts
interface ResultHeroProps {
  archetype: Archetype

}
```

Layout (top to bottom within a centered container, max-w-2xl mx-auto):
1. **Illustration placeholder**: A styled `div` (w-28 h-28 rounded-full mx-auto) with the archetype's color variant. Use a Lucide icon inside (e.g., `Heart`, `Shield`, `Compass`, `Star`, `Sprout`, `BookOpen`, `Sun`, `Trophy`, `Users` — one per archetype). Map each archetype ID to a specific icon and distinct warm background color (amber-100, rose-100, emerald-100, sky-100, violet-100, orange-100, teal-100, pink-100, indigo-100). This is a placeholder that can be swapped to `<img>` later.

2. **Archetype name**: Large, prominent text. Use `text-3xl sm:text-4xl font-bold text-stone-800`. Display as "Your Parenting Archetype:" (smaller text above) then the archetype name (e.g., "The Steady Anchor") on a new line.

3. **Secondary blend mention**: Subtle text below the name: `"with shades of {secondaryArchetypeName}"` in `text-stone-500 text-sm`.

4. **Tagline**: The archetype's tagline in `text-lg text-stone-600 italic` below the name.

5. **Teaser summary**: 2-3 sentence summary composed from the archetype's `foundationalPatterns.headline` and the first theme from `foundationalPatterns.themes[0]`. Format: "{headline} {themes[0]}" — this provides a clinically grounded teaser that maps to the PDF chapter preview. Wrap in `text-stone-700 text-base leading-relaxed`.

**Animation**: Add a CSS fade-in-up animation to `globals.css`:
```css
@keyframes fade-in-up {
  from {
    opacity: 0;
    transform: translateY(16px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```
Add a Tailwind v4 `@utility` for `animate-fade-in-up` with `animation: fade-in-up 0.8s ease-out both`.

Apply staggered animation delays to child elements:
- Illustration: `animation-delay: 0s`
- Archetype name: `animation-delay: 0.2s`
- Secondary blend: `animation-delay: 0.3s`
- Tagline: `animation-delay: 0.4s`
- Teaser summary: `animation-delay: 0.6s`

Each child gets `opacity-0 animate-fade-in-up` with inline `style={{ animationDelay }}` for the stagger (since Tailwind doesn't support arbitrary animation-delay values).

Wire `ResultHero` into `ResultShell.tsx` — render it as the first section after data load.
